# Release Notes — v0.1.0b

**Дата:** 2026-04-22
**Тип:** первый публичный бета-релиз

---

## Обзор

Первый публичный бета-релиз RedLycoris. Платформа переходит из internal-разработки в стадию бета-тестирования с внешними командами.

Основные области: агрегация findings от 6 типов сканеров, 7 источников обогащения, приоритизация и RBAC.

---

## Что нового

### Импорт и парсеры

Поддерживаемые форматы при импорте через `POST /api/v1/import`:

| Формат | Парсер | Примечания |
|--------|--------|------------|
| SARIF 2.1.0 | `SARIFParser` | Semgrep, CodeQL, OpenGrep |
| Trivy JSON | `TrivyParser` | SCA (OS packages, libraries) |
| Checkov JSON | `CheckovParser` | IaC misconfigurations |
| Gitleaks JSON | `GitleaksParser` | Secrets в коде |
| TruffleHog v3 | `TruffleHogParser` | NDJSON и array-формат |
| Generic JSON | `GenericParser` | Любой кастомный источник |

### Обогащение

Все синхронизации работают асинхронно через Redis Streams, не блокируют API.

Интервалы обновления:
- NVD: каждые 2 часа (инкрементальный)
- EPSS: ежедневно в 01:00 UTC
- CISA KEV: каждые 6 часов
- БДУ ФСТЭК: еженедельно
- OSV: ежедневно
- CWE: ежемесячно
- CPE: еженедельно

### Приоритизация

```
priority_score =
    cvss_base        × 0.30
  + epss × 100       × 0.25
  + (10 if kev)      × 0.20
  + (5  if bdu)      × 0.10
  + recency          × 0.10
  + exposure         × 0.05
```

`recency = 10 × exp(−days_since_published / 365)` — экспоненциальное затухание за год.

Materialized view обновляется каждые 5 минут.

### Производительность

Результаты нагрузочного тестирования (`docs/ops/load-testing.md`):

- Импорт 100k findings: < 60 секунд
- GET /api/v1/findings (50 записей, cursor): < 50 мс p95
- GET /api/v1/dashboard/stats: < 200 мс p95 (materialized view)
- Обогащение 100k findings: ~ 30 минут (3 воркера, NVD rate limit)

### Безопасность

- Rate limit на `/api/v1/auth/login`: 5 попыток / 15 мин per IP+email
- Cookie `rl_session`: `HttpOnly; Secure; SameSite=Strict`
- Backend запускается от non-root пользователя (uid 1000)
- Все пароли — bcrypt

## Инструкция по установке

Первый релиз — upgrade notes не применимы.

```bash
git clone https://github.com/nefrit0n/red-lycoris.git
cd red-lycoris
cp env.example .env
# Обязательно заполнить: POSTGRES_PASSWORD, BOOTSTRAP_ADMIN_EMAIL, BOOTSTRAP_ADMIN_PASSWORD
./scripts/build.sh
docker compose up -d
```

## Известные проблемы

- **OSV sync**: загружает все экосистемы без фильтрации; при медленном канале (< 5 Мбит/с) возможен таймаут первой синхронизации. Обходной путь: увеличить таймаут или запустить `scripts/sync-all.sh` вручную.
- **Jira**: двунаправленная синхронизация не реализована — только однонаправленный push planned для 0.2.0.
- **SBOM**: CycloneDX / SPDX не поддерживаются — planned для 0.2.0.
